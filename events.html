CodeSense

“Awaken your coding senses!”

This event tests the candidate's sense of coding. Various keywords of a programming language are given a specific weight. Participant has to code the given problem and ensure the code acquires minimum weight according to the specified weights.

Rules:

  1. This is an offline event in which candidates can participate individually.
  2. The participants can code in any of the given languages: C, C++, Java, and Python.
  3. Participants will be given two problems to solve.
    - The first will be judged on the basis of time complexity of the program
    - The second will be judged on the basis of the weightage of the program as per the specified weights.
  4. Weight for every keyword used in code will be predefined.
  5. The participant has to code such that his/her code has minimum weight.
  6. Participant having the least time complexity for the first solution and minimum weighted code for the second will considered winner.
  7. In case of a tie the code with the minimum time complexity will be considered better.

The 1st and 2nd place holder will be awarded prize money.

Exeplore

“Explore the way to get through”

This event consists of two rounds. In the first round, participants have to answer the question realted to video which will be shown before questioning round. Second round will be conducted on java based app designed by team members. Participats need to solve puzzels on this app.

* This event is open for students of all the branches. No coding skills required!

Specification and format:

  1. This event requires a team of 2 members.
  2. Event will be conducted in 2 stages.
  3. First stage of event will consist of a preliminary round consisting of 15 questions based on a movie clip of duration 5-10 minutes. After that each participating team will be given a time of 15 minutes to solve the questions.
  4. The questions will be either MCQs or one word answer type.
  5. Second stage of event will be a puzzle solving event.
  6. Participants have to explore the way out by solving each puzzle correctly that will take them to another level of puzzle on a computer based interface.
  7. The one who solves the puzzles fastest or who scores highest will be declared as the winner

Rules:

  1. Maximum of 2 members are allowed per team. Individual participation is also allowed.
  2. The event is strictly based on efficient and intuitive thinking.
  3. Based on the performance of the teams in Preliminary Round, top 10-15 teams will be selected for the Final round.
  4. In the final round a team of two will be provided with only 1 computer to trace the way out.
  5. No external help will be provided.
  6. In any circumstances, judge’s decision will be final and binding to all.

Judging Criterion:

  1. The team that solves all the puzzles first will be winner.
  2. In case no team is able to solve all puzzles, team with the maximum score will be the winner.

The 1st and 2nd place holder will be awarded prize money.

CodeSwap

“Code..., stop and switch!”

This is an offline event in which students need to participate in a team of 2 members to solve given programming problems by writing a code. The interesting thing is that – the 2 members need to write code on separate system at same time and, they have to switch their seats after a specified time. This will test their capability to work in team and understanding other person’s code.

Rules:

  1. Strictly 2 members are allowed per team.
  2. The event analyses your problem solving ability and team working spirit.
  3. This event will consist of only one round.
  4. Both team members will be provided with different computers.
  5. The participants will be provided with 2 questions
    6.   - One they have to solve together
      - Second to be solved separately
  6. Participants will be given two problems, first problem will be based on simple mathematics, and team members can solve it together.
  7. Second problem will be based on some algorithm which is to be solved separately by both members and swap seats after 10 minutes and complete each other’s code.
  8. The team that solves the problem first, correctly will be winner.
  9. The participants can code in any of the given languages: C, C++, Java, and Python.

The 1st and 2nd place holder will be awarded prize money.

HackHunt

“Hack and Hunt!”

This event tests programming skills of participants. In this event, there are three programs which the participant has to find output.

In the first two questions:

the participant must detect the maximum number of inputs that does not satisfy the specified logic of the program given as a executable file in a specified time.

In the time allotted to a programmer, the programmer must understand the program thoroughly in order to detect the source for which the outputs corresponding to the inputs given to program by the programmer are logically incorrect.

The programmer detecting the maximum number of inputs that behave incorrectly according to the logic of the program will achieve the score equal to the number of outputs he/she detects.

The score achieved by the programmer will be added to the score which he/she will get from the third i.e. last question of the event.

The third question will be a program and the participant has to detect its output. The output detected will be a cipher and the participant has to decode the cipher using a sample given to them. The time taken by the participant to solve the question will be given the score accordingly.

The combined score of 1st, 2nd and 3rd question would be the final score achieved by the participant.

The participant with the highest score will be considered as the winner.

The 1st and 2nd place holder will be awarded prize money.
